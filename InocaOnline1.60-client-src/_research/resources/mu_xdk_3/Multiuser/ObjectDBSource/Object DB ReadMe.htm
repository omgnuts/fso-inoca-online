Introduction

This package contains sample source code for the Shockwave Multiuser Server's Object DB xtra. This is the xtra that implements the object database server commands.

This code is provided as an example of a server xtra for your education and modification. Macromedia is not able to answer technical support questions concerning the use of this code.

 

Code Summary

The xtra uses the Sequiter Codebase engine as a back-end database. This database contains information about the attributes, users, applications, and the actual stored information. Since this is a 3rd-party licensed product, the actual library and include files are not part of this distribution. If you are interested, you may purchase the Codebase library from Sequiter.

The main server/xtra interface is in the SrvExtension.cpp file.

The back-end databases contain the following information:

Applications - maps between an application name and ID number
AppData - maps between an data ID and application ID
AttrList - maps between an attribute name and ID
Attributes - has info about attributes (owner ID, size, etc)
IDTable - keeps track of the internal ID numbers
Players - maps user ID &amp; application ID to a player ID
RawData - has actual attribute data
Users - has User's name, internal ID and password

The ODB*.cpp files are abstract implementation of these tables, while the DBCB_*.cpp files are the actual tables using the Codebase library.

This table architecture and lookup methods are excellent candidates for improvement and replacement with a modern back-end database such as an SQL system.


Compiling

To build the object database, use the instructions provided with the server XDK. Some of the files included here are the same as the common ones from the XDK; they are provided to make a more complete package.

For compiling, the following values are #defined for all builds:

WINDLL
_WINDLL

WINVER=0x0400
_WINVER=0x0400
WINDOWS
_WINDOWS
WIN32
_WIN32

WIN32_LEAN_AND_MEAN


For debug builds, the following additional values are #defined:

DEBUG
_DEBUG
TEST
_TEST
XDEBUG=1


For release builds, the following additional values are #defined:

RELEASE
_RELEASE
NDEBUG
_NDEBUG
XDEBUG=0
