Multiuser Server XDK 3.0 ReadMe

Welcome to the Multiuser Server XDK release. The goal of this XDK is to enable you to write server Xtras that extend the functionality of the Multiuser server. The XDK is designed to work with version 8.X of the Director XDK and version 3.0 of the server.

This release supercedes any previous release of the multiuser XDK. Backward compatibility is generally supported, but any new APIs clearly are not supported in earlier versions of the server.

Contents of the Multiuser XDK

Common - Headers, code and resource files used by server xtras.
Docs - ReadMe and Functional Specification.
MarsLogger - A sample Xtra Project.
MoaXDK - Defines server interfaces and callbacks.
Stl - Standard header files that are not required but may be used by server Xtras.
Template - The core files required to build server Xtras.
Xtras - folder into which the Sample Xtra builds. Builds of the Mac and Windows MarsLogger Xtras - Logger and Logger.x32 - are provided.


Required Supporting Files

The server xtras depend on some of the files contained in the Director 8 XDK. Your xtra projects and workspaces should link to the D8 XDK's Include, Lib and Runtime directories. Xtras also require linking to Multiuser MOAXDK.

To install the Multiuser XDK, extract the contents of the ZIP into Example folder. Specify a new Multiuser folder in the Extract To field in the WinZip Extract dialog.

Notes on the Sample Files and Projects

MarsLogger - A server xtra designed to log server events. Intended as a ready-to-build project that will illuminate the server-xtra architecture and enable a quick start in designing your own xtras. This project uses some of the header files in the STL that is provided, but the Multiuser server extension Xtras do not require the STL.
Template - Use these to build your own xtras from the ground up. How-To steps are provided in the Tempalte How-To file.

Known Issues

--There is some inconsistency in the indexing - some methods use zero-based, others use one-based.
--The SendMessageToGroup method has a formal parameter of recipient. This is in error but passing in a NULL value will produce an error. The code will be changed in a future release. In the meantime, any string will be accepted as the actual parameter.
-- The MarsLogger is provided with some code not enabled and mainly server as an extensible example Xtra.

Other Notes

The supported development environments are Microsoft Visual C++ version 6.0 and MetroWerks CodeWarrior Pro version 5.0 (IDE version 4.01). The projects in the XDK are designed to open in those IDEs.
